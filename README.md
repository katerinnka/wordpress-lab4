# Лабораторная работа №4. Разработка плагина для WordPress

## Описание лабораторной работы

В рамках лабораторной работы №4 был разработан пользовательский плагин для WordPress под названием **USM Notes**.

Цель работы — освоить расширяемую модель данных WordPress:
- создание Custom Post Type (CPT);
- регистрация пользовательской таксономии;
- работа с метаданными (meta fields);
- создание шорткода для отображения данных.

## Функциональность плагина

Плагин добавляет новый раздел **“Заметки” (Notes)** в WordPress с возможностью:

- создавать заметки;
- назначать приоритет (High / Medium / Low);
- указывать дату напоминания;
- фильтровать заметки по параметрам;
- выводить заметки на страницу через шорткод.


## Установка и запуск
1. Скопировать папку плагина в: wp-content/plugins/usm-notes
2. Зайти в админ-панель WordPress  
3. Перейти в раздел **Plugins (Плагины)**  
4. Активировать плагин **USM Notes**


## Структура реализации

### 1. Custom Post Type (CPT)

Создан новый тип записей:
- **Notes (Заметки)**
- поддерживает:
  - заголовок
  - текст
  - автора
  - изображение
2. Зайти в админ-панель WordPress  
3. Перейти в раздел **Plugins (Плагины)**  
4. Активировать плагин **USM Notes**


### 2. Пользовательская таксономия

Добавлена таксономия:
- **Priority (Приоритет)**
- значения:
  - High
  - Medium
  - Low


### 3. Метаполе (Meta Box)

Добавлено поле:
- **Дата напоминания (Due Date)**

Особенности:
- обязательное поле
- нельзя выбрать дату в прошлом
- реализована валидация
- используется nonce для безопасности

- ### Возможности:
- фильтрация по приоритету
- фильтрация по дате
- вывод всех заметок (если параметры не указаны)

- ## Тестирование

Для проверки были созданы:
- 5-6 заметок
- разные приоритеты
- разные даты

Результат:
- фильтрация работает корректно

## Ответы на контрольные вопросы

### 1. Разница между таксономией и метаполем

**Таксономия** используется для группировки данных (например: категории, приоритеты).  
**Метаполе** - это дополнительная информация о записи (например: дата).

Пример:
- Priority → таксономия  
- Due Date → метаполе  


### 2. Зачем нужен nonce

Nonce используется для защиты данных от:
- CSRF атак
- несанкционированного изменения данных

Если его не проверять — любой может отправить данные и изменить записи.


### 3. Важные аргументы register_post_type()

- `public` → делает записи доступными на сайте  
- `supports` → определяет доступные функции (title, editor и т.д.)  
- `has_archive` → включает страницу архива  
 Эти параметры влияют на UX и отображение контента.
